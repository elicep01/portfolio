SQL Indexes - why your SELECT got 100x faster

Backend

An index is a pre sorted map from key to rows. Without it, the DB scans every row, O(n). With an index, it jumps via a tree, about O(log n).

Start by indexing columns you filter or join on; keep indexes lean, and watch write costs since indexes slow inserts and updates.
