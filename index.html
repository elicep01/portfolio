About

I am a Master's student in Electrical and Computer Engineering at UW-Madison focused on Machine Learning and Backend systems. I design and ship reliable services and ML features that are simple, fast, and observable. Previously at Siemens, I worked across low code and core product fronts. I care about clean APIs, data correctness, and evaluation first ML so models actually improve products.

What I do
  • Backend - design clear REST APIs and services in Python or Node, with Postgres and Redis.
  • Reliability and performance - tracing, metrics, profiling, caching, and SQL tuning.
  • ML that ships - small efficient models in PyTorch or scikit learn, offline eval first, then iterate.
What I did
  • Ex-Siemens Digital Industries Software - 4 years shipping production features across web and mobile.
  • Built UI components and developer tools (React and React Native, GoJS, PDFTron) and contributed to Teamcenter Active Workspace frontend.
  • Led performance minded engineering and automation testing with cross team collaboration.
Now / Learning
  • Backend depth - API versioning, design docs, request tracing, Postgres indexing, Redis caching patterns.
  • ML engineering - dataset quality checks, reproducible experiments, strong baselines, evaluation dashboards.
